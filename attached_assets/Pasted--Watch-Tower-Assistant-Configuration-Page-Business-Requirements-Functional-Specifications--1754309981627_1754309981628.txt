# Watch Tower Assistant Configuration Page
## Business Requirements & Functional Specifications

---

## üéØ Page Overview

**Purpose**: Central configuration interface for personalizing the Watch Tower AI assistant that monitors, learns, and engages on behalf of users across digital platforms.

**Core Function**: Transform a generic AI into a personalized intelligence companion through comprehensive configuration options.

---

## üèóÔ∏è Page Structure

### **Main Layout**
- **Header Section**: Page title and description
- **Tabbed Configuration Interface**: 4 primary tabs
- **Save Button**: Persistent save action at bottom
- **Task Creation Modal**: Overlay for creating scheduled tasks

### **Tab Organization**
1. **Personality** - AI voice, tone, and content preferences
2. **Intelligence** - Data sources and monitoring setup
3. **Tasks** - Automated research and action scheduling
4. **Engagement** - Social media automation and boundaries

---

## üéõÔ∏è Tab 1: Personality Configuration

### **Assistant Identity**
- **Assistant Name**: Text input for AI companion name
- **Communication Style**: Dropdown selection
  - Professional & Analytical
  - Casual & Friendly  
  - Formal & Academic
  - Creative & Expressive

### **Voice & Tone Settings**
- **Tone Preference**: Slider (Conservative ‚Üî Bold)
- **Expertise Level**: Slider (Beginner-Friendly ‚Üî Expert)

### **Content Preferences**
Checkbox grid for preferred content types:
- Industry Insights
- Technical Deep Dives
- Opinion Pieces
- Research Summaries
- Personal Stories
- How-To Guides

### **Interests & Expertise**
- **Add Interface**: Text input + Add button
- **Dynamic List**: Display added interests with remove options
- **Default Examples**: AI, Machine Learning, Data Science

---

## üß† Tab 2: Intelligence Sources

### **Social Media Monitoring**
Four input fields in 2x2 grid:
- **LinkedIn Profile URL**: Full LinkedIn profile URL
- **Twitter/X Handle**: Username with @ symbol
- **Facebook Page**: Facebook page URL
- **Instagram Handle**: Username with @ symbol

### **RSS Feeds & News Sources**
- **Add Interface**: URL input + "Add Feed" button
- **Feed List**: Display added feeds with remove options
- **Default Examples**: TechCrunch AI Feed, MIT Technology Review

### **Key People to Monitor**
- **Add Interface**: Name/handle input + "Add Person" button
- **Person List**: Display monitored people with remove options
- **Default Examples**: Andrej Karpathy, Yann LeCun

### **Intelligence Briefing Schedule**
- **Morning Briefing Time**: Time picker (default: 08:00)
- **Evening Summary Time**: Time picker (default: 18:00)
- **Briefing Format**: Dropdown selection
  - Executive Summary (2-3 key insights)
  - Detailed Report (5-7 insights with context)
  - Quick Highlights (1-2 sentences each)

---

## üìÖ Tab 3: Tasks & Automation

### **Task Creation**
- **"Add Task" Button**: Opens modal for new task creation
- **Task Modal Components**:
  - Task Name input
  - Frequency buttons (Once, Daily, Weekly, Monthly, Yearly)
  - Execution Time picker
  - Instructions textarea
  - Create Task button

### **Active Tasks Display**
List of existing tasks showing:
- **Task Name** with status indicator (green=active, gray=inactive)
- **Frequency Badge**: Visual frequency indicator
- **Instructions Preview**: First line of task instructions
- **Action Buttons**: Edit, pause, delete options

### **Relevance Scoring Settings**
- **Quality Threshold**: Slider (Low Quality ‚Üî High Quality Only)
- **Recency Weight**: Slider (Timeless Content ‚Üî Breaking News Focus)

---

## ‚ù§Ô∏è Tab 4: Engagement & Automation

### **Automation Level Control**
- **AI Autonomy Slider**: 5-level scale with descriptions
  1. Manual Approval (All actions require approval)
  2. Suggested Actions (AI suggests, user approves)
  3. Semi-Autonomous (AI can like/share, approval for comments)
  4. Mostly Autonomous (AI engages freely, daily summary)
  5. Fully Autonomous (Complete automation, weekly oversight)

### **Content Creation Settings**
- **Posts Per Day**: Dropdown (1-2, 3-4, 5-6, Custom schedule)
- **Post Visibility**: Dropdown (Always Public, Draft for Review, Private by Default)
- **Approval Prompt Template**: Textarea for customizing AI approval requests

### **Engagement Strategy**
Checkbox options for AI engagement behaviors:
- Auto-like relevant posts
- Share with commentary
- Reply to comments on my posts
- Initiate conversations
- Engage with network posts
- Follow suggested accounts

### **Safety & Boundaries**
- **Content Guidelines**: Textarea for defining content restrictions
- **Safety Options**: Checkboxes for safety measures
  - Fact-check before sharing
  - Avoid controversial topics
  - Never engage in debates
  - Maintain professional tone

## üíæ Data Requirements

### **Configuration Data Structure**
```javascript
{
  personality: {
    name: "Assistant name",
    communicationStyle: "Selected style",
    tone: 70,                    // 0-100 slider value
    expertise: 80,               // 0-100 slider value
    contentPreferences: [...],   // Array of selected types
    interests: [...]             // Array of user interests
  },
  intelligence: {
    socialMedia: {
      linkedin: "URL",
      twitter: "@handle",
      facebook: "URL", 
      instagram: "@handle"
    },
    rssFeeds: [...],            // Array of RSS URLs
    keyPeople: [...],           // Array of names/handles
    briefingSchedule: {
      morning: "08:00",
      evening: "18:00",
      format: "Selected format"
    }
  },
  tasks: {
    qualityThreshold: 80,       // 0-100 slider value
    recencyWeight: 30,          // 0-100 slider value
    activeTasks: [...]          // Array of task objects
  },
  engagement: {
    autonomyLevel: 3,           // 1-5 scale
    postsPerDay: "3-4 posts",
    postVisibility: "Draft for Review",
    approvalPrompt: "Template text",
    engagementStrategy: [...],  // Array of enabled strategies
    contentGuidelines: "Guidelines text",
    safetySettings: [...]       // Array of safety options
  }
}
```
