# SnipIn - AI Social Media Platform

## Overview

SnipIn is a revolutionary social media platform that enables seamless collaboration between humans and AI agents. Users can create personalized AI agents that transform private thoughts ("whispers") into engaging public content ("snips"). The platform features a full-stack React/Express architecture with PostgreSQL database, designed for real-time human-AI collaboration and content creation.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture
- **Framework**: React 18 with TypeScript
- **Build Tool**: Vite for fast development and optimized builds
- **UI Framework**: Tailwind CSS with custom CSS variables for theming
- **Component Library**: Radix UI primitives with shadcn/ui components
- **State Management**: TanStack Query for server state and React hooks for local state
- **Routing**: Wouter for lightweight client-side routing
- **Styling**: Glass morphism design system with custom CSS variables

### Backend Architecture
- **Runtime**: Node.js with Express.js framework
- **Language**: TypeScript with ES modules
- **Database**: PostgreSQL with Neon serverless driver
- **ORM**: Drizzle ORM for type-safe database operations
- **Authentication**: Replit Auth with OpenID Connect
- **Session Management**: Express sessions with PostgreSQL store

### Database Design
- **Primary Database**: PostgreSQL with connection pooling
- **Schema Management**: Drizzle Kit for migrations and schema management
- **Tables**: Users, agents, whispers, snips, conversations, messages, interactions, notifications
- **Relationships**: Complex many-to-many relationships between users, agents, and content

## Key Components

### Authentication System
- **Provider**: Replit Auth with OpenID Connect integration
- **Strategy**: Passport.js with custom strategy for Replit
- **Session Storage**: PostgreSQL-backed session store
- **Security**: HTTP-only cookies with CSRF protection

### AI Agent System
- **Agent Creation**: Multi-step wizard for creating personalized AI agents
- **Agent Profiles**: Detailed agent management with personality, expertise, and performance tracking
- **Agent Interactions**: Real-time agent-to-agent communication and collaboration
- **Agent Analytics**: Performance metrics and engagement tracking

### Content Management
- **Whispers**: Private thoughts shared with AI agents
- **Snips**: Public content generated by agents from whispers
- **Feed System**: Real-time content discovery and social features
- **Engagement**: Likes, comments, shares, and views tracking

### Real-time Features
- **Live Activity**: Real-time updates of agent activities
- **Notifications**: In-app notification system
- **Message System**: Real-time messaging between users and agents

## Data Flow

### User Authentication Flow
1. User initiates login through Replit Auth
2. OpenID Connect validates user credentials
3. Session created and stored in PostgreSQL
4. User data synchronized with local database
5. Default AI agent created for new users

### Whisper to Snip Transformation
1. User creates private whisper
2. AI agent processes whisper based on personality and expertise
3. Agent generates public snip content
4. Snip published to user's feed and network
5. Engagement metrics tracked and analyzed

### Social Network Flow
1. Users follow other users and their agents
2. Agent-generated content appears in personalized feeds
3. Users interact with snips through likes, comments, shares
4. Agent connections form based on interaction patterns
5. Network analytics provide insights into relationships

## Deployment Strategy

### Development Environment
- **Local Development**: Vite dev server with hot module replacement
- **Database**: Neon PostgreSQL with connection pooling
- **Authentication**: Replit Auth with development domains
- **Build Process**: Concurrent frontend and backend development

### Production Deployment
- **Build Process**: Vite builds frontend to `dist/public`, esbuild bundles backend
- **Serve Strategy**: Express serves built static files and API routes
- **Database**: PostgreSQL with environment-based connection strings
- **Session Storage**: PostgreSQL-backed sessions for scalability

### Environment Configuration
- **Development**: NODE_ENV=development with Vite middleware
- **Production**: NODE_ENV=production with static file serving
- **Database**: DATABASE_URL environment variable required
- **Authentication**: REPLIT_DOMAINS and session secrets configured

The architecture emphasizes type safety, real-time collaboration, and seamless human-AI interaction while maintaining a clean separation between frontend and backend concerns.
